# i18n Conventions

## Type-Safe Translation Keys

All translation keys are typed via `I18nKey` — a union type auto-generated from scope JSON files.

- **Generated file**: `src/i18n/i18n-keys.generated.ts` (per app)
- **Generator**: `i18n-keygen:keys` executor
- **Auto-runs**: `i18n` target is a dependency of `build` and `serve`
- **Cross-language sync**: the generator validates that all language files have identical key structures

## Full-Path Keys

Generated keys always use fully-qualified paths including the scope prefix. Templates use `'core.appTitle'` (Transloco) or `'core:appTitle'` (i18next) — never just `'appTitle'`.

## Framework Wrappers

Each framework wraps its i18n library with a typed layer. Never use the underlying i18n library directly in app code.

| Framework | Wrapper | Underlying |
|-----------|---------|------------|
| Angular (Transloco) | `I18nPipe`, `I18nService` | `TranslocoService` |
| React (i18next) | `useI18n` hook | `useTranslation` |
| Vue (i18next) | `useI18n` composable | `useTranslation` |

## Translation Key Naming

- Semantic namespaces: `form`, `table`, `status`, `actions`, `dialog`.
- Root-level keys only for metadata: `pageTitle`, `emptyState`.
- Dot notation for nesting: `nav.home`, `status.pending`.

## Scope Separators

| Library | Separator | Example |
|---------|-----------|---------|
| Transloco | `.` | `core.appTitle` |
| i18next | `:` | `core:appTitle` |

## Dead Key Cleanup

When removing a feature, always remove its translation keys from all locale files. Run `npx nx run-many -t i18n` after removal to verify.
